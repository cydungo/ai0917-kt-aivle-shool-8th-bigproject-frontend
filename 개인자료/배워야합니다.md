# 배워야 합니다: 핵심 기술 요약

이 문서는 프론트엔드 및 AI 초보자가 이 프로젝트를 이해하고 개발하는 데 필요한 핵심 개념과 순서를 요약한 것입니다.

## 1. 프론트엔드 핵심 (React + Vite + TypeScript)

### 개발 순서 (Workflow)

1. **UI 컴포넌트 만들기**: 화면에 보이는 버튼, 입력창 등을 만듭니다. (`src/app/components/`)
2. **페이지 만들기**: 컴포넌트들을 조립하여 하나의 화면(페이지)을 만듭니다. (`src/app/pages/`)
3. **라우팅 연결**: URL 주소에 따라 페이지가 보이도록 연결합니다. (`src/app/routes/` 또는 `App.tsx`)
4. **API 연결**: 백엔드 서버에서 데이터를 가져오거나 보냅니다. (`src/app/services/`)

### 꼭 알아야 할 개념

- **컴포넌트 (Component)**: 레고 블록처럼 재사용 가능한 UI 조각. (예: `<Button />`, `<Header />`)
- **Props**: 부모 컴포넌트가 자식에게 전달하는 데이터. (예: `<Button color="red" />`에서 `color`)
- **State (useState)**: 컴포넌트 내부에서 변하는 데이터. (예: 입력창의 내용, 모달의 열림/닫힘 상태)
- **Effect (useEffect)**: 화면이 처음 그려질 때나 특정 데이터가 바뀔 때 실행되는 코드. (API 호출 등)

## 2. 데이터 관리 (TanStack Query + Axios)

### 왜 쓰는가?

- **Axios**: 브라우저가 서버와 통신하기 위한 도구 (fetch보다 편리함).
- **TanStack Query (React Query)**: 서버에서 가져온 데이터를 **캐싱(저장)**하고, **동기화(최신 상태 유지)**하는 도구. "로딩 중", "에러 발생", "데이터 성공" 상태를 쉽게 관리해줍니다.

### 사용 패턴

```typescript
// 데이터 가져오기 (GET)
const { data, isLoading } = useQuery({
  queryKey: ['todos'], // 데이터의 고유 이름
  queryFn: fetchTodos, // 데이터를 가져오는 함수
});

// 데이터 변경하기 (POST, PUT, DELETE)
const mutation = useMutation({
  mutationFn: postTodo, // 데이터를 변경하는 함수
  onSuccess: () => {
    // 성공 시 할 일 (예: 목록 새로고침)
  },
});
```

## 3. 스타일링 (Tailwind CSS + Shadcn/UI)

- **Tailwind CSS**: CSS 파일을 따로 만들지 않고, 클래스 이름으로 스타일을 지정합니다.
  - 예: `<div className="p-4 bg-blue-500 text-white">` (패딩 4, 배경 파랑, 글자 흰색)
- **Shadcn/UI**: 미리 만들어진 고퀄리티 컴포넌트 모음입니다. 복사해서 가져다 쓰면 됩니다.
  - **모달 (Dialog)**: 사용자에게 중요한 알림을 띄우거나 입력을 받을 때 사용합니다. `open` 속성으로 제어합니다.

## 4. 아키텍처 및 AI (초보자 관점)

### CSR (Client Side Rendering)

- 이 프로젝트는 **CSR** 방식입니다.
- **작동 원리**: 브라우저가 처음에 빈 HTML과 자바스크립트 뭉치를 다운로드받고, 브라우저에서 화면을 그립니다.
- **장점**: 화면 전환이 깜빡임 없이 부드럽습니다 (앱처럼). 백엔드 서버 부하가 적습니다.

### 백엔드 구조 (Spring Boot + PostgreSQL)

- **Spring Boot**: 자바 기반의 튼튼한 백엔드 프레임워크.
- **PostgreSQL**: 데이터를 저장하는 관계형 데이터베이스.
- **pgvector**: AI 검색(유사도 검색)을 위해 벡터 데이터를 저장하는 플러그인.

### AI 연결

- 프론트엔드는 AI 모델을 직접 돌리지 않습니다 (브라우저가 너무 느려짐).
- 대신, **API 요청**을 통해 백엔드나 별도의 AI 서버에 데이터를 보내고, 결과만 받아서 화면에 보여줍니다.

## 6. 추천 도구 (VS Code Extensions)

마크다운(Markdown) 파일을 예쁘게 보고 싶다면 아래 확장 프로그램을 설치해보세요!

    - 에러가 난 코드 줄에 바로 빨간 글씨로 무슨 에러인지 알려줍니다. (초보자 강추!)

    - 타입스크립트의 복잡한 에러 메시지를 사람이 읽기 쉽게 예쁘게 보여줍니다.
