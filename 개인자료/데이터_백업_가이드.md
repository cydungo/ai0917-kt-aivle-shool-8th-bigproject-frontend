## ☁️ 방법 1: 클라우드 동기화 (OneDrive / Google Drive)

> **💡 긴급 팁 (이미 쓰고 있다면?)**
> 혹시 파일이 사라졌나요? **구글 드라이브(또는 OneDrive) 휴지통**을 먼저 확인해보세요!
> (우리의 소중한 파일이 거기서 낮잠을 자고 있을 수도 있습니다. 😴)

윈도우(Windows)를 쓰고 있으니까 **OneDrive**가 이미 있을 거야. 이걸 활용하는 가장 스마트한 방법(심볼릭 링크)을 알려줄게.
이 방법은 **파일을 이동하지 않고도** 클라우드랑 자동으로 연결해줘. 완전 마법 같지? ✨

### 1. 명령 프롬프트(CMD) 열기

1. `윈도우 키`를 누르고 `cmd` 검색.
2. **[관리자 권한으로 실행]** 클릭! (이거 중요해 ⭐)

### 2. 마법의 주문(명령어) 입력하기

아래 명령어를 복사해서 붙여넣기만 하면 돼. (경로는 네 컴퓨터 환경에 맞게 수정했어!)

**OneDrive를 쓰는 경우:**

```cmd
mklink /J "%UserProfile%\OneDrive\개인자료_백업" "%UserProfile%\Desktop\projects\big_project\frontend\ai0917-kt-aivle-shool-8th-bigproject-frontend\개인자료"
```

**Google Drive를 쓰는 경우:**
(보통 `G:` 드라이브나 `내 드라이브` 폴더가 있어. 경로 확인 필요!)

```cmd
mklink /J "G:\내 드라이브\개인자료_백업" "%UserProfile%\Desktop\projects\big_project\frontend\ai0917-kt-aivle-shool-8th-bigproject-frontend\개인자료"
```

> **성공 메시지:** `교차점을 만들었습니다...` 라고 뜨면 성공! 🎉
> 이제 `개인자료` 폴더에 파일을 넣으면, 자동으로 OneDrive에도 똑같이 생겨.

## 🔐 방법 2: 나만의 비밀 금고 (Private Git Repo)

개발자라면 **Git**으로 관리하는 것도 멋지지!
메인 프로젝트랑은 별개로, `개인자료` 폴더 안에서만 작동하는 **작은 Git**을 만드는 거야.

1. **GitHub** 웹사이트에 가서 **New Repository** 클릭.
2. 저장소 이름 대충 짓고 (예: `my-secret-study`), **Private(비공개)** 체크! ✅ (이거 안 하면 다 보여!)
3. 터미널을 열고 아래 명령어 입력:

```bash
# 1. 개인자료 폴더로 이동
cd 개인자료

# 2. 새로운 Git 시작
git init

# 3. 파일 담기
git add .
git commit -m "내 소중한 자료 백업 완료!"

# 4. 내 비밀 저장소랑 연결 (GitHub에서 주소 복사해오기)
git remote add origin https://github.com/네아이디/저장소이름.git

# 5. 발사! 🚀
git push -u origin main
```

## 🤔 무엇을 백업해야 할까?

1.  **`개인자료/` (필수 ⭐⭐⭐)**
    - 이건 `.gitignore`에 등록돼서 메인 프로젝트(GitHub)에는 안 올라가.
    - 그러니까 **무조건 따로 백업**해야 해! 위 방법 중 하나를 꼭 써.

2.  **`가이드라인/` (선택)**
    - 이게 만약 **"팀원들이랑 같이 봐야 하는 규칙"**이라면? 👉 그냥 메인 프로젝트에 `commit` 해서 올리는 게 좋아. (팀원들도 봐야 하니까!)
    - 만약 **"나 혼자 몰래 보는 비급"**이라면? 👉 `개인자료` 폴더 안으로 옮기거나, 위 방법으로 같이 백업해.
